# 《塔陨：异步终局》Demo二次试玩复测（玩法体现 / 数值 / UI操作）

## 1. 复测范围
- 方式：Playwright MCP 实机操作
- 流程：从开局推进至 `9/9`
- 核查维度：
  - Roguelike + 自走棋 + 爬塔 + 背包构筑 + 吃鸡淘汰 是否在局内可感知
  - 数值节奏是否合理
  - UI与操作链路是否顺畅

## 2. 五大玩法“是否有体现”结论

### 2.1 Roguelike（有体现，强度中）
- 可见随机路径节点：事件、商店、战斗、营地、精英、层主
- 可见随机事件结果（示例日志：事件战斗胜/败，奖励与损失波动）
- 结论：**已体现随机决策分叉**，但“随机事件文本与后果解释”仍偏薄

### 2.2 自走棋（有体现，强度中）
- 商店购买单位后自动上阵，棋盘可见单位职业定位（frontline/backline）提示
- 发生自动战斗并产出战力对比结果（日志有战力值）
- 结论：**阵容构筑和自动战斗已体现**，但手动站位与克制反馈还不够“可学习”

### 2.3 爬塔（有体现，强度高）
- 楼层明确从 `1/9 -> 9/9` 推进
- 每层都存在“本层路径 + 封盘结算 + 进入下一层”的循环
- 结论：**爬塔主循环完整且可持续推进**

### 2.4 背包构筑（有体现，强度中）
- 道具购买后进入背包，支持整理放置
- 有“战前加成摘要”（如层结算回复+8）
- 结论：**基础构筑已打通**，但“形状/邻接/朝向”的策略深度在当前可视反馈里仍偏弱

### 2.5 吃鸡淘汰（有体现，强度中偏高）
- 存活估计人数从 `30 -> 28 -> 26 -> 21 -> 18 -> 14 -> 12 -> 10 -> 6 -> 3`
- 异步PVP结算持续驱动淘汰收敛
- 结论：**淘汰压力存在且可感知**，但玩家个人“排名/淘汰风险预警”表达还可加强

## 3. 数值合理性评估

### 3.1 观测到的关键轨迹
- 早期（2~4层）：
  - HP：`83 -> 78 -> 80`
  - Gold：`11 -> 21 -> 21`
  - Alive：`28 -> 26 -> 21`
- 中后期（5~9层）：
  - HP：`69 -> 71 -> 73 -> 75 -> 77 -> 79`
  - Gold：`41 -> 59 -> 76 -> 96 -> 111 -> 126`
  - Alive：`18 -> 14 -> 12 -> 10 -> 6 -> 3`

### 3.2 结论
- **经济膨胀偏快**：5层后金币增长陡峭，容易削弱“买与不买”的权衡感
- **血线压力后期偏轻**：中后期HP连续回升，紧张感更多来自“人数收缩”而非生存边缘
- **徽记惩罚存在感不足**：本次长流程内徽记长期维持 `3`，吃鸡“失误代价”不够突出

### 3.3 数值建议
- 适度下调后4层固定金币注入，增加与“战斗质量/事件选择”挂钩权重
- 提升7层后失败惩罚斜率（HP或徽记至少一项更有压迫）
- 为终局层增加“风险倍率”提示，强调收官阶段的生存压力

## 4. UI与操作合理性评估

### 4.1 优点
- “稳健线/贪收益线”与“建议动作序列”显著降低了新手决策门槛
- 探索/事件按钮有风险收益提示，策略意图更清楚
- 路径面板能快速理解本层结构与已走进度

### 4.2 问题
- `9/9` 终局出现长时间停留在“异步匹配与结算”阶段（流程稳定性风险）
- 购买后单位常“自动上阵”，候战区经常为空，可能让玩家误以为购买失败
- 日志信息量大但“最新关键结论”不够醒目，复盘效率一般

### 4.3 UI/交互建议
- 终局层加“结算等待超时提示 + 重试/回退策略”
- 购买单位后给强反馈条：`已自动上阵至 X 号位`
- 日志区增加“关键事件高亮”（PVP胜负、徽记变化、淘汰阈值）

## 5. 综合判定
- **五大玩法均已在游戏内体现**，其中“爬塔循环”最完整。
- 当前核心改进点不在“有没有玩法”，而在：
  - 终局结算稳定性
  - 数值后期膨胀控制
  - 自走棋与背包策略反馈可读性
- 建议先做“终局稳定 + 数值曲线修正 + 关键反馈高亮”三项，收益最高。
