## 2. 项目一句话

**Roguelike + 自走棋 + 爬塔 + 背包构筑 + 吃鸡淘汰** 的弱联网异步对战：30 人同 RunID，层内 PVE 构筑，层末封盘提交快照，服务端权威异步 PVP 结算，徽记归零淘汰，最后存活者吃鸡。

---

## 3. 核心循环（单层）

```
自由探索(T1) → 路径收缩(T2) → 封盘提交(T3) → 异步匹配与战斗 → 层结算(发奖/扣血/扣徽记) → 下一层
```

- **体验目标**：高策略构筑、低实时操作压力、强终局紧张感；**结果可解释性**为首要体验目标（输赢原因可学习）。
- **关键接口**：`SubmitFloorSnapshot` → `RequestAsyncBattle` → `GetBattleResult` → `AckFloorResult`。

---

## 4. Demo 验证结论（截至 2026-03-19）

- **五大玩法**：Roguelike、自走棋、爬塔、背包构筑、吃鸡淘汰均在局内可感知；爬塔主循环完整度最高。
- **已落地体验优化**：稳健/贪收益策略线、本层建议动作序列、探索/事件收益与风险提示、战前加成摘要、系统建议卡、路径节点与动作强约束、日志关键高亮去重。
- **待优先解决**：
  1. **战斗结果可解释性**：失败归因卡、快照对战事件轴（关键帧）。
  2. **数值**：后期经济膨胀控制、7 层后失败惩罚斜率、终局风险倍率提示。
  3. **流程与 UI**：终局层「异步匹配与结算」长时间停留（稳定性）、购买单位后「已自动上阵」强反馈、关键事件高亮。
