# 《塔陨：异步终局》GDD v2（30人同场、弱联网、异步PVP+PVE）

## 1. 项目定位
- 类型：Roguelike + 自走棋 + 爬塔 + 背包构筑 + 吃鸡淘汰
- 目标体验：高策略构筑、低实时操作压力、强终局紧张感
- 联网目标：不做强实时同步；只在关键节点联网结算
- 单局人数：30人同房间（同种子、同规则、同时间窗）

## 2. 核心定义（异步“同场”）
- 同场不等于实时同屏，而是同一 `RunID` 下共享：
  - 相同塔地图随机种子
  - 相同层规则与缩圈时序
  - 相同奖励池和Boss轮换
- 对抗方式：镜像快照战（Ghost PvP），由服务器权威重算结果
- 玩法结构：层内PVE构筑 + 层末异步PVP淘汰 + 全局缩圈推进

## 3. 单局流程（35分钟目标）

### 3.1 开局阶段（2分钟）
- 玩家进入房间并锁定职业
- 服务器下发：`RunID`、`Seed`、赛季词缀、首层规则
- 玩家选择落点（影响首层资源倾向）

### 3.2 每层循环（约4分钟/层，8层+终局）
- 探索期（PVE）：打野怪、事件、商店、背包整理、自走棋阵容调整
- 封盘期（15秒）：提交当前构筑快照
- 匹配期（异步PVP）：服务器分配对手快照并权威结算
- 结算期：发放奖励、扣徽记、更新排名并进入下一层

### 3.3 终局层（3~5分钟）
- 规则强化（高压缩圈 + Boss干扰 + 连续遭遇）
- 根据存活、徽记、终局积分判定冠军（吃鸡）

## 4. 五大玩法融合落地

### 4.1 Roguelike（肉鸽）
- 随机节点、随机事件、随机遗物、随机商店池
- 局内成长重置，局外保留轻度Meta解锁（新职业、新遗物池）

### 4.2 自走棋
- 棋盘 `7x4`，单位1~5费，三合一升星
- 职业/种族羁绊（首发建议 `6职业 x 6种族`）
- 战斗全自动，玩家只做前置构筑与站位

### 4.3 爬塔（杀戮）
- 每层难度递增，每3层层主，终局层总Boss干扰
- 路线分叉：高风险高收益节点与保守节点并存

### 4.4 背包英雄
- 背包初始 `10x6`，可扩展到 `12x8`
- 装备有形状、朝向、邻接触发
- 背包词条影响自走棋战斗行为（开场盾、低血触发、击杀回蓝等）

### 4.5 吃鸡
- 使用“时间圈 + 路线圈”替代实时空间圈
- 每层有提交截止与路径关闭机制，拖延即受罚
- 生命徽记为0即淘汰，最后存活者吃鸡

## 5. 关键系统规格

### 5.1 背包系统
- 物品分类：武器、防具、功能、触发器、遗物
- 核心机制：
  - 邻接加成（上下左右）
  - 朝向加成（与前后排关联）
  - 组合配方（战后自动合成）
- 示例：
  - `火药包` 邻接远程武器：+12%暴击率
  - `稳压核心` 邻接法系道具：技能冷却 -8%

### 5.2 自走棋战斗系统
- 伤害类型：物理、法术、真实
- 标签系统：召唤、持续伤害、爆发、控制、护盾
- 目标选择：最近、最低血、最高威胁（按单位AI模板）

### 5.3 异步PVP快照系统
- 快照内容：单位列表、星级、装备、背包布局、遗物、站位、Buff状态
- 快照有效期：10分钟（防止过旧数据被反复匹配）
- 匹配策略：同层优先、战力分桶、最近未交手优先
- 补位策略：在线不足时用同段位快照池

### 5.4 缩圈与淘汰系统
- 每层三段时间：
  - `T1` 自由探索
  - `T2` 路径收缩（关闭部分节点）
  - `T3` 强制封盘（未提交扣徽记）
- 初始徽记：3
- 扣减来源：PVP失败、超时未提交、圈外停留（路径封锁后逗留）

### 5.5 经济系统
- 每层收入：基础金币 + 连胜/连败 + 利息（10/20/30/40/50阈值）
- 三资源：
  - 金币：买单位与刷新
  - 齿轮：背包改造
  - 秘能：遗物重铸、净化诅咒
- 反滚雪球：
  - 低徽记补给
  - 后位保护匹配（短时）
  - 商店保底刷新机制

## 6. 数值初版（可直接开平衡）
- 玩家数：30
- 层数：8 + 终局层
- 单局时长：32~38分钟
- 单层时长：220秒（探索170 + 封盘15 + 结算35）
- 初始血量：100
- 徽记：3
- 单位池：24（MVP） -> 48（首赛季）
- 道具池：40（MVP） -> 120（首赛季）
- 遗物池：30（MVP）

## 7. 技术架构（弱联网）

### 7.1 客户端职责
- 本地PVE模拟、背包编辑、UI与表现
- 记录操作日志与随机种子使用点
- 到结算点上传快照

### 7.2 服务器职责（权威）
- 房间编排、时间窗推进、匹配、PVP重算
- 奖励结算、排行榜、淘汰状态管理
- 异常检测与反作弊判定

### 7.3 关键接口（示意）
- `CreateRun / JoinRun`
- `GetRunConfig`
- `SubmitFloorSnapshot`
- `RequestAsyncBattle`
- `GetBattleResult`
- `AckFloorResult`
- `GetRunLeaderboard`
- `FinishRun`

## 8. 反作弊方案（必做）
- 服务端权威战斗：客户端不得上报战斗结果，只上报构筑快照
- 日志哈希链：关键行为串联哈希，检测篡改与回滚
- 种子校验：关键随机事件基于服务端种子重放
- 异常模型：非法高收益路径、异常掉落密度、非人类操作频率告警
- 断线策略：允许层内离线，封盘前必须联网提交，否则按超时惩罚

## 9. UI/UX主流程
- 大厅：赛季词缀、房间报名倒计时
- 层内：左侧塔层路径、中央探索区域、右侧背包与商店
- 封盘：15秒“锁定构筑”提示
- 对战：自动战斗回放 + 关键触发时间轴
- 结算：排名、徽记变化、奖励与下一层预告

## 10. MVP范围（12~16周）
- 功能范围：
  - 12人同房（先验证） -> 30人扩容
  - 4层塔 + 1终局层
  - 24单位、40道具、30遗物
  - 异步PVP镜像战 + 基础反作弊
- 验证指标：
  - D1 > 35%
  - 单局完成率 > 60%
  - 平均对局时长 28~36分钟
  - 举报率 < 1.5%
- 扩容门槛（上30人）：
  - 结算服务 P95 < 1.2s
  - 快照失败率 < 0.5%

## 11. 风险与预案
- 风险：异步对战临场感不足
  - 预案：战斗回放演出、复仇标记、同层宿敌系统
- 风险：高策略导致新手流失
  - 预案：推荐构筑、一键整理背包、前3层新手保护
- 风险：时间窗压力过高
  - 预案：提供一次性延时券道具，降低挫败
- 风险：强Build垄断
  - 预案：赛季词缀轮换、热修平衡、道具池动态权重

## 12. 版本迭代建议
- `v0.1`：单机PVE + 本地自动战斗，验证“背包 x 羁绊”手感
- `v0.2`：异步快照战与层结算闭环
- `v0.3`：12人灰度，经济与缩圈平衡
- `v1.0`：30人正式上线 + 赛季化
